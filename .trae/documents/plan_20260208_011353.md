# Implementasi Tombol Finish Sesi & Pause

Saya akan memisahkan fungsi "Pause" dan "Finish" agar lebih jelas dan fungsional. Saat ini tombol "Stop" melakukan keduanya (stop timer & call API finish), yang membingungkan.

## Rencana Implementasi

1.  **Modifikasi Logic Frontend (`race-master.blade.php`)**:
    -   Memecah `stopRace` menjadi dua fungsi:
        -   `pauseRace()`: Hanya menghentikan timer client-side (`timer.running = false`) tanpa memanggil API finish.
        -   `finishRace()`: Menghentikan timer, memanggil API `/finish`, dan mengarahkan ke tab Results. Akan ada konfirmasi `confirm()` sebelum eksekusi.
    -   Update `startRace()`: Tetap sama (resume timer).

2.  **Update UI Control Button**:
    -   **Tombol Play/Pause (Toggle)**:
        -   Jika Timer Mati (`!running`): Tombol **Play (Hijau)**.
        -   Jika Timer Jalan (`running`): Tombol **Pause (Kuning)**.
    -   **Tombol Finish (Baru)**:
        -   Tombol **Bendera Finish (Merah)** terpisah.
        -   Selalu muncul jika waktu berjalan (`elapsed > 0`).
        -   Klik -> Konfirmasi -> Finalize Sesi -> Pindah ke Results.
    -   **Tombol Reset**: Tetap ada untuk reset total.

## UI Layout Baru
```
[ Play/Pause ]  [ Finish ]  [ Reset ]  [ Camera ]  [ Toggle Scanner ]
```
- **Play/Pause**: Besar, indikator utama.
- **Finish**: Penting, tapi butuh konfirmasi.
- **Reset**: Sekunder.

Ini akan menjawab kebutuhan user: "belum ada button finish sesi" (sekarang ada dedicated button) dan "sekarang masih ada pause saja" (fungsi pause sekarang eksplisit).
